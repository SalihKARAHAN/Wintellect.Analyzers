WINTELLECT002: Call Debug.Assert with message parameters

TypeName

CallAssertMethodsWithMessageParametersAnalyzer

CheckId

WINTELLECT002

Category

Usage

Breaking Change

Non Breaking
Cause

A call was made to the Debug.Assert overload that only takes one parameter, the condition.

Rule Description

Specifying the assertion condition as a message will show up in the assertion output and make debugging easier.

Best practices dictate that for the message parameter you should use the text in the condition parameter as the message parameter. That way you see what is triggering the assertion.

How to Fix Violations

To fix a reported error, add the message parameter to the call to Debug.Assert or use the Code Fix provider to automatically fix.

When to Suppress Warnings

Do not suppress a warning from this rule.

Example

The following example shows a method, BadExample, that violates the rule because it uses only the single parameter overload of Debug.Assert. In GoodExample the message parameter is properly used.

C# 
using System;
using System.Diagnostics;
class ExampleLibrary
{
   public void BadExample(Int32 level)
   {
      Debug.Assert(i > 0);
   }
   
   public void GoodExample(Int32 level)
   {
      Debug.Assert(i > 0,"i > 0");
   }
}
