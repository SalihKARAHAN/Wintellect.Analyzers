WINTELLECT007: Assemblies have CopyrightAttribute Analyzer

TypeName

 AssembliesHaveCopyrightAttributeAnalyzer

CheckId

WINTELLECT007

Category

Wintellect.Design

Breaking Change

Non Breaking
Cause

The assembly being analyzed does not have a CopyrightAttribute set at the assembly level. Either it is missing or the string is empty.

Rule Description

Copyrights, even in open source software, are very important to protect your intellectual property and rights. All assemblies you build need to have the CopyrightAttribute set at the assembly level.

How to Fix Violations

Somewhere in the assembly code, generally in a AssemblyInfo.cs file, ensure you have the AssemblyCopyrightAttribute set to a valid string.

When to Suppress Warnings

Do not suppress a warning from this rule.

Example

The following example how to set the AssemblyCopyrightAttribute.

C# 
using System.Reflection;
[assembly: AssemblyCopyright("Copyright © 2015 Wintellect")]
