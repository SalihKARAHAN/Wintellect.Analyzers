WINTELLECT003: if and else statements must have braces

TypeName

IfAndElseMustHaveBracesAnalyzer

CheckId

WINTELLECT003

Category

Formatting

Breaking Change

Non Breaking
Cause

A C# if or else statement was found that does not use braces to deliniate the expression.

Rule Description

Code readability is one of the keys to high maintenance and when the statement after an if or else condition are not surrounded by braces, the code is harder to read. Additionally, dangling expressions can also lead to bugs because adding a second statement to the expression is not part of the expression unless surrounded by curly braces.

This beginnings of this rule was presented by Kevin Pilch-Bisson at Europe 2014. We just finished it off and made it work for else clauses as well.

How to Fix Violations

Insert the braces around the expression or use the Code Fix provider to automatically fix.

When to Suppress Warnings

Do not suppress a warning from this rule.

Example

The following example shows two if statements. The first incorrectly uses braces. The second shows how the code should be formatted.

C# 
using System;
using System.Diagnostics;

class ExampleLibrary
{

   public Boolean BadExample(Int32 level)
   {
      if (i > 0) return true;
      else return false;
   }
   
   public Boolean GoodExample(Int32 level)
   {
      if (i > 0)
      {
         return true;
      }
      else
      {
          return false;
      }
   }
}
