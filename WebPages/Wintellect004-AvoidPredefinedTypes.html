WINTELLECT004: Avoid Predefined Types Analyzer

TypeName

 AvoidPredefinedTypesAnalyzer

CheckId

WINTELLECT004

Category

Usage

Breaking Change

Possbily
Cause

You are using a primitive predefined type when you should be using the explicit type.

Rule Description

According to Jeffrey Richter's CLR via C#, it's better to use the explicity Framework Class Library (FCL) types instead of the C# primitives. See the Primitive, Reference, and Value Types chapter for an in-depth discussion.

How to Fix Violations

Use the built in Code Fix provider to convert the predefined primitive to the appropriate FCL type.

When to Suppress Warnings

Do not suppress a warning from this rule.

Example

The following example shows an in correct method and a fixed method.

C# 
using System.Reflection;
using System;
using System.Collections.Generic;
using System.Collections.Immutable;
using System.Threading;
using System.Threading.Tasks;

namespace Example
{
    public class SomeClass
    {
        public bool IncorrectType()
        {
            return false;
        }

        public Boolean CorrectType()
        {
            return true;
        }

    }
}
