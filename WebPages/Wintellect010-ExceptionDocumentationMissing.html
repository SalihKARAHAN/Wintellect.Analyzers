WINTELLECT010: Exception XML document tag is missing

TypeName

ExceptionDocumentationMissingAnalyzer

CheckId

WINTELLECT010

Category

Documentation

Breaking Change

Non Breaking
Cause

A method throws an exception, but the exception is not documented in the XML Doc Comments.

Rule Description

Any time you have a direct throw in a public method and you do not document that throw, you make life much harder for anyone using your code. The exceptions you throw are critical data and if they are not documented anyone calling your method is forced to read your source code, or worse, use a decompilation tool to determine what exceptions are thrown.

A direct throw is where you specifically are calling throw in your code. You are not supposed to document every possible exception for any and all code you might call. The intent of the exception XML doc comment tag are direct throws.

How to Fix Violations

To fix a reported error, add the exception tag to your documentation.

When to Suppress Warnings

Do not suppress a warning from this rule.

Example

The following example shows a method, BadExample, that violates the rule because it does not specify the exception tag. In FixedExample the exception tab is properly used.

C# 
using System;
using System.Diagnostics;
namespace ExampleLibrary
{
    /// <summary>
   /// Prints out the data specified.
   /// </summary>
   /// <param name="loop">
   /// The number of items to display
   /// </param>
   public void BadExample(Int32 loop)
   {
      if (i==0)
      {
         throw new InvalidArgumentException("loop");
      }
      Int32 i = 0;
      throw while (i < loop )
      {
         i++;
         Console.WriteLine("i={0}",i.ToString());
      }
   }
     
   /// <summary>
   /// Prints out the data specified.
   /// </summary>
   /// <param name="loop">
   /// The number of items to display
   /// </param>
   /// <exception cref="InvalidArgumentException>
   /// Thrown if <paramref name="loop" is zero.
   /// </exception>
   public void FixedExample(Int32 loop)
   {
      if (i==0)
      {
         throw new InvalidArgumentException("loop");
      }
      Int32 i = 0;
      throw while (i < loop )
      {
         i++;
         Console.WriteLine("i={0}",i.ToString());
      }
   }
}
