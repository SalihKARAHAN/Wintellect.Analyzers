WINTELLECT011: SuppressMessageAttribute missing Justification parameter Analyzer

TypeName

SuppressMessageMissingJustificationParameterAnalyzer

CheckId

WINTELLECT011

Category

Documentation

Breaking Change

Non Breaking
Cause

A SuppressMessageAttribute declaration is missing the Justification parameter

Rule Description

When suppressing a message from tools like Code Analysis, you need to justify why you are suppressing that message. While you can include a comment, there is no way to ensure the comment explaining why the tool us reporting a false positive or the tool recommendation should be ignored. Using the Justification parameter ensures the reasoning for suppression is supplied.

If the assembly, class, or method has the GeneratedCodeAttribute or DebuggerNonUserCodeAttributes, this rule will not analyze those classes to avoid issues with generated code.

How to Fix Violations

To fix a reported error, add the Justification parameter with an explanation as to why you are suppressing the tool error.

When to Suppress Warnings

Do not suppress a warning from this rule.

Example

The following example shows a class, XoXBadExample, that violates the rule because it does not supply the Justification parameter. In WiXTask the Justification parameter is properly used.

C# 
using System;
using System.Diagnostics;
using System.Diagnostics.CodeAnalysis;
namespace ExampleLibrary
{
   [SuppressMessage("Microsoft.Naming",
                    "CA1706:ShortAcronymsShouldBeUppercase")]
                    public class XoXBadExample
   {
   }

   [SuppressMessage("Microsoft.Naming",
                    "CA1706:ShortAcronymsShouldBeUppercase",
                    Justification="WiX is the official name of the tool.")]
                    public class WiXTask
   {
   }
}
